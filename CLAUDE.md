# Правила работы с проектом

## 📚 Obsidian Vault — Память проекта

Проект использует Obsidian Vault как постоянную память: `Obsidian/TwitchDownloaderVault/`

### Обязательные правила:

1. **Начало каждой сессии**: Прочитай `Obsidian/TwitchDownloaderVault/index.md` перед началом работы.
   Это даёт контекст о структуре проекта без повторного изучения кода.

2. **Навигация через wikilinks**: Все заметки связаны через `[[ссылки]]`.
   От `index.md` переходи к нужным разделам через ссылки.

3. **Обновление при изменениях**: Если ты редактируешь файл и:
   - Меняешь функциональность существующего метода/компонента → обнови соответствующую заметку в vault
   - Добавляешь новый метод/файл/компонент → создай или дополни заметку в `modules/` или `api/`
   - Удаляешь что-то → отметь в заметке как устаревшее или удали заметку

4. **Формат записи изменений** (в `changelog/YYYY-MM-DD.md`):
   ```
   ## [filename] — [краткое описание изменения]
   - Что изменилось: ...
   - Почему: ...
   - Связанные файлы: [[ссылка]]
   ```

5. **Структура заметок**: Большие темы разбивай на несколько файлов,
   связывая их с родительским через `[[Parent]]` в начале дочерней заметки.
